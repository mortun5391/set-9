# Практическое исследование алгоритмов сортировки строк

Автор: **Халилбеков Х. А.**

Проект содержит генератор строковых тестов, шесть алгоритмов сортировки, класс для эмпирических замеров, самостоятельные реализации для задач A1* и краткий отчёт.

## Реализованные алгоритмы

1. `QUICKSORT` - стандартный comparison-based quicksort по целым строкам.
2. `MERGESORT` - стандартный comparison-based mergesort по целым строкам.
3. `STRING_QUICKSORT` - тернарный string quicksort.
4. `STRING_MERGESORT_LCP` - string mergesort с сохранением LCP соседних строк.
5. `MSD_RADIX_SORT` - MSD radix sort без переключения.
6. `MSD_RADIX_SORT_WITH_STRING_QS` - MSD radix sort с переключением на string quicksort при размере подмассива `< 74`.

Для `QUICKSORT` и `STRING_QUICKSORT` опорным выбирается элемент с индексом `l + (r - l) / 2`.

## Тестовые данные

`StringGenerator` использует строки длины от 10 до 200 и массивы максимального размера 3000. Для каждого `n = 100, 200, ..., 3000` берётся префикс максимального массива.

Создаются наборы:

- `random` - случайный;
- `reverse` - обратно отсортированный;
- `nearly_sorted` - сортированный массив с одной перестановкой соседней пары в каждом блоке из 100 строк;
- `common_prefix` - дополнительный стресс-тест с общим префиксом длины 70.

В условии указано 74 символа, поэтому завершающая точка после перечисления специальных символов трактуется как пунктуация; генератор использует специальные символы `!@#%:;^&*()-`.

## Метрики

- `avg_time_us`, `min_time_us`, `median_time_us` - время в микросекундах по 11 запускам;
- `char_comparisons` - количество посимвольных сравнений;
- `char_inspections` - дополнительная метрика чтений символов для корректной интерпретации MSD Radix Sort.

Каждая сортировка автоматически сверяется с `std::sort`.

## Сборка и воспроизведение полного CSV

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
./build/string_sort_benchmark --out data/results.csv --inputs data/generated_arrays --runs 11 --seed 20260524
python3 tools/plot_results.py data/results.csv figures
```

## Опубликованные материалы
