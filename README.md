# Практическое исследование алгоритмов сортировки строк

Автор: **Халилбеков Х. А.**

Публичный репозиторий: **https://github.com/mortun5391/set-9**

Проект содержит генератор строковых тестов, шесть алгоритмов сортировки, класс для эмпирических замеров, исходные CSV-данные эксперимента и Markdown-отчёт. Графики строятся скриптом `tools/plot_results.py`; DOCX/PDF-версия отчёта приложена в комплекте для сдачи.

## Реализованные алгоритмы

1. `QUICKSORT` - стандартный comparison-based quicksort по целым строкам.
2. `MERGESORT` - стандартный comparison-based mergesort по целым строкам.
3. `STRING_QUICKSORT` - тернарный string quicksort.
4. `STRING_MERGESORT_LCP` - string mergesort с сохранением LCP соседних строк.
5. `MSD_RADIX_SORT` - MSD radix sort без переключения.
6. `MSD_RADIX_SORT_WITH_STRING_QS` - MSD radix sort с переключением на string quicksort при размере подмассива `< 74`.

Для `QUICKSORT` и `STRING_QUICKSORT` опорным выбирается элемент с индексом `l + (r - l) / 2`, как требуется для корректного сравнения вариантов.

## Тестовые данные

`StringGenerator` использует строки длины от 10 до 200 и массивы максимального размера 3000. Для каждого значения `n = 100, 200, ..., 3000` берётся префикс максимального массива. Создаются наборы:

- `random` - случайный;
- `reverse` - обратно отсортированный;
- `nearly_sorted` - сортированный массив с одной перестановкой соседней пары в каждом блоке из 100 строк;
- `common_prefix` - дополнительный стресс-тест: все строки имеют общий префикс длины 70.

В условии указано 74 символа. Поэтому завершающая точка после перечисления специальных символов трактуется как пунктуация; генератор использует специальные символы `!@#%:;^&*()-`.

## Метрики

- `avg_time_us`, `min_time_us`, `median_time_us` - время в микросекундах по 11 запускам.
- `char_comparisons` - число сравнений двух реальных символов. Сравнение с концом строки не учитывается.
- `char_inspections` - дополнительная диагностическая метрика: число чтений символов. Для MSD radix sort `char_comparisons = 0`, потому что он распределяет строки по корзинам, а не сравнивает пары символов; поэтому для него информативна именно эта дополнительная колонка.

Каждая сортировка автоматически проверяется сравнением с `std::sort`; при несовпадении эксперимент останавливается.

## Сборка и запуск

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
./build/string_sort_benchmark --out data/results.csv --inputs data/generated_arrays --runs 11 --seed 20260524
python3 tools/plot_results.py data/results.csv figures
```

## Структура файлов

- `include/`, `src/` - классы `StringGenerator`, `StringSortTester` и реализации алгоритмов;
- `submissions/` - самостоятельные файлы `A1q.cpp`, `A1m.cpp`, `A1r.cpp`, `A1rq.cpp`;
- `data/results.csv` - исходные результаты измерений;
- максимальные входные массивы воспроизводятся генератором с фиксированным seed `20260524`;
- `data/summary_at_3000.csv` - выборка результатов при `n = 3000`;
- `figures/` - построенные графики в полном комплекте; при клонировании репозитория они генерируются командой из раздела «Сборка и запуск»;
- `report/REPORT.md` и `report/submission_info.txt` - текст отчёта и место для ID посылок.
