# Отчёт по исследованию сортировок строк

Автор: Халилбеков Х. А.

## Эксперимент

Исследованы шесть алгоритмов: обычные Quicksort и Mergesort, тернарный String Quicksort, String Mergesort с LCP, MSD Radix Sort и гибрид MSD Radix Sort с переключением на String Quicksort для подмассива размером менее 74.

Использованы четыре набора строк: случайный, обратно отсортированный, почти отсортированный и дополнительный стресс-тест с общим префиксом длины 70. Размер массива изменяется от 100 до 3000 с шагом 100. Время усредняется по 11 запускам. Результат каждого запуска сверяется со стандартной сортировкой C++.

## Итоги при n = 3000

| Набор | Лучший алгоритм | Среднее время, мкс | Ускорение относительно лучшего стандартного |
|---|---|---:|---:|
| random | String Quicksort | 457.6 | 1.76x |
| reverse | MSD + String Quicksort | 182.8 | 2.22x |
| nearly_sorted | MSD + String Quicksort | 192.5 | 1.71x |
| common_prefix | String Quicksort | 1036.6 | 5.99x |

## Выводы

Стандартные сортировки заметно проигрывают на строках с длинными совпадающими префиксами, поскольку повторно просматривают одинаковые начальные части строк. Тернарный String Quicksort показал лучший результат на случайном и префиксном наборах. String Mergesort с LCP особенно сильно уменьшает число посимвольных сравнений: на наборе `common_prefix` оно составляет около 10.8% от обычного Mergesort. Гибрид MSD со String Quicksort стал лучшим на обратно и почти отсортированных данных.

Подробные численные результаты для размера 3000 расположены в `data/summary_at_3000.csv`. Полный файл результатов, графики и оформленная версия отчёта входят в комплект сдачи.
